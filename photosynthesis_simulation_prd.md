# 제품 요구사항 정의서 (PRD): 광합성 환경 요인 탐구 시뮬레이션 앱

## 1. 프로젝트 개요
* **프로젝트명:** 광합성 환경 요인 탐구 시뮬레이션 (Antigravity 기반)
* **목표:** 중/고등학교 생명과학 교과 과정 중 '광합성에 영향을 미치는 환경 요인(빛의 세기, 이산화 탄소 농도, 온도)'을 학습할 수 있는 가상 실험 시뮬레이션 앱 개발 및 배포.
* **타겟 사용자:** 과학 탐구 활동을 수행하는 학생 및 교사.
* **플랫폼:** Antigravity를 활용한 웹/모바일 반응형 앱.

## 2. 화면 구성 및 UI/UX 요구사항
앱은 크게 좌측(시뮬레이션 조작부)과 우측(결과 데이터 및 그래프)으로 나뉩니다. (모바일의 경우 상/하 배치)

### 2.1. 좌측 패널 (조작 및 시각화)
* **탐구 탭 (Tab Navigation):** 
  * 탐구 1 (빛), 탐구 2 (CO₂), 탐구 3 (온도) 탭 제공.
  * 탭 전환 시, 조작 변인을 제외한 나머지 통제 변인은 최적값으로 자동 초기화 및 고정됨.
* **환경 요인 슬라이더 (Range Input):**
  * 빛의 세기 (0~100%, 10단위)
  * CO₂ 농도 (0~100%, 10단위)
  * 온도 (0~50°C, **1도 단위**)
* **시각화 영역 (Canvas / Animation):**
  * 수조 속 검정말(수초)과 빛을 비추는 전등 그래픽.
  * 빛의 세기에 따라 전등 그래픽의 밝기 변화.
  * 측정 중일 때 산소 기포가 발생하는 애니메이션 출력.
* **측정 버튼:**
  * '산소 생성량 측정하기' 버튼. 누르면 4초간 측정 상태(애니메이션) 유지 후 결과 도출.

### 2.2. 우측 패널 (데이터 및 시각화)
* **데이터 초기화 버튼:** 현재 탭의 실험 데이터를 모두 삭제.
* **실험 결과 활동지 (Data Table):**
  * 열 구성: [조작 변인값] | [1회] | [2회] | [3회] | [평균 (ml/분)]
  * 동일한 조작 변인값에서 최대 3회까지 측정값을 누적하여 기록.
  * 3회 측정이 완료되어야 평균값이 계산되어 표출됨.
* **변화 그래프 (Line Chart):**
  * X축: 현재 활성화된 탭의 조작 변인 (빛, CO₂, 온도)
  * Y축: 산소 생성량 (평균값, ml/분)
  * 데이터 테이블에서 **3회 측정이 완료된(평균이 구해진) 데이터 포인트만** 꺾은선 그래프로 렌더링.

## 3. 핵심 기능 및 비즈니스 로직 (Antigravity 상태 관리)

### 3.1. 변인 통제 로직
* **탐구 1 (빛 조작):** CO₂=100%, 온도=25°C 고정. 빛(0~100) 조작.
* **탐구 2 (CO₂ 조작):** 빛=100%, 온도=25°C 고정. CO₂(0~100) 조작.
* **탐구 3 (온도 조작):** 빛=100%, CO₂=100% 고정. 온도(0~50) 조작.

### 3.2. 산소 생성량 산출 모델 (수학적 모델)
앱 내부 Calculation 노드(또는 Custom Function)를 통해 다음 수식을 구현합니다.
1. **빛 요인 (f_L):** `light / (light + 15)` (포화 곡선)
2. **CO₂ 요인 (f_C):** `co2 / (co2 + 15)` (포화 곡선)
3. **온도 요인 (f_T):**
   * 온도 <= 35 일 때: `(temp / 35)^2` (기하급수적 증가)
   * 온도 > 35 일 때: `MAX(0, 1 - (temp - 35) / 10)` (효소 변성으로 인한 급감)
4. **최종 생성량 (ml/분):**
   * `Base Rate = 2.5 * MIN(f_L, f_C) * f_T` (최소량의 법칙 적용)
   * `Noise = 랜덤값(-0.1 ~ +0.1)` (실험 오차 구현)
   * `Result = MAX(0, Base Rate + Noise)` (소수점 2자리 반올림)
   * 단, 빛=0 또는 CO₂=0 이면 무조건 0 반환.

### 3.3. 측정 및 데이터 기록 프로세스
1. 사용자가 변인 설정 후 [측정하기] 버튼 클릭.
2. `isMeasuring` 상태를 `true`로 변경.
3. 시각화 영역에 계산된 산소 생성량에 비례하는 기포 애니메이션 트리거.
4. **4초 대기 (Timeout 4000ms)**.
5. 대기 종료 후 측정된 결과값을 현재 탭의 데이터 리스트에 추가.
   * 동일한 X값(조작 변인값)이 이미 존재하면, 측정 횟수 배열(`m: []`)에 결과값을 `push`.
   * 이미 3회가 꽉 찼다면 마지막 3회차 데이터를 덮어쓰기.
6. `isMeasuring` 상태를 `false`로 변경.

## 4. Antigravity 배포 시 유의사항
* **State Management (상태 관리):** `activeTab`, `light`, `co2`, `temp`, `experiments`(JSON Array 형태), `isMeasuring` 상태 변수를 전역 또는 페이지 레벨에서 선언해야 합니다.
* **Chart Widget 연동:** Antigravity의 기본 Line Chart 위젯을 사용하되, Query/Filter 기능을 이용해 `experiments` 배열 내에서 "측정 횟수가 3회인 데이터의 평균값"만 추출하여 차트의 데이터 소스로 바인딩해야 합니다.
* **Animation 처리:** Antigravity의 조건부 렌더링(Conditional Visibility)을 사용하여 `isMeasuring == true`일 때 Lottie 애니메이션 또는 CSS 기포 컴포넌트가 나타나도록 설정합니다.
